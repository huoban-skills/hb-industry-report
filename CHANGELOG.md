# Changelog

## v0.1.1 — 2026-07-13

**产出物**
- 产出改为按行业名建文件夹三件套：HTML 报告 + 正文 markdown + `figures/` 独立 SVG（文件内嵌配色，便于复用到飞书/PPT）。

**框架与写法**
- 摘要精简为两段三四句：要素（是什么/阶段/痛点/诉求）是暗线，阅读感是主线，不套固定句式，措辞句式不得复用上一份报告。
- 1.1 边界 callout 改为「报告主角与范围」，主角先行（第一句写明报告站在谁的视角），末尾指向图 3。
- 图 2 产品全景按行业类型分变体：服务/金融型（要素×生命周期×订单家族）、实体产品型（产品解剖含成本占比 × 价值链 × 品类矩阵）。
- 2.3 节名用词随行业（资金上下游 / 产业链上下游），明确图 2 与图 3 分工：拆开看物 vs 排开看角色，成本大头 ≈ 强势上游。
- 3.3 痛点根因收敛为 2–4 个、比喻自拟，去掉"三座大山"硬模板。

**图规范**
- 链条全景图主角块必须带「▼ 本报告主角」金色标注，figcaption 说明深色实心块含义。
- 排版选型：链条层级 ≥4 用竖版（两侧标资金/服务流向），流向复杂（钱/货/票分层）用横版双层；监管方虚线框。

**通用化**
- 去服务业偏置：需求分层按行业选切法（组织 vs 经办人 / 渠道 vs 终端 / 决策者 vs 使用者）、特色图分行业举例、术语表增加工艺与材料分组、素材提炼清单覆盖生产流程与成本结构。
- 修正过时的"摘要三段"自检项。

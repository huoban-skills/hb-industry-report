# 逐节写作指南

每节按此指南填充 template.html。整体基调：**先讲故事建立认知，再讲生意，最后讲业务**；术语首次出现就地解释；图 > 表 > 字——图 1–4 是骨架必备，其余位置**能用图讲清就不要用大段文字**，随内容自由加图（机制、结构、对比都值得成图），文字只写重点结论和必要的场景化描述。全文 8000–12000 字为宜。

## 封面与摘要

- **主标题**：纯行业名（如"城投定融行业""央国企电商采购供货行业"）。
- **副题**：破折号开头，一句话点出行业本质（"—— 地方隐性债务的形成、兑付与化解"）。
- **目录**：两级——章 + 小节（附录列词条分组）。小节 h3 带 `id="s章-节"`、附录 h4 带 `id="gl-N"`，目录链接与之对应；宽屏（≥1380px）目录固定在左侧栏（无边框轻量样式、细滚动条）并随滚动高亮当前小节，窄屏退化为正文内目录块。模板已含样式与文末脚本，按实际标题生成列表即可。
- **摘要**（`.abstract`，两段、三四句话，宁短勿长）：任务是让人 20 秒明白"这是个什么行业、难在哪、要什么"。这四个要素是暗线，行文以阅读感为主线——基于行业本身自然成文，像向同事口头介绍一个行业那样说话，不套任何固定句式；细节（数字、环节链、机制）都留给正文。

## 一、行业背景

**1.1 行业定义**：一句话定义 → 生活化类比（加粗，这是全文记忆锚点）→ `callout`「报告主角与范围」，**主角先行**：第一句写明"本报告站在谁的视角"（行业链条上的哪个角色是主角，附主体画像），再写讨论范围（含什么业务）、排除项（不含什么），最后带同类主体变体。不要用"行业边界"这类读者看不懂的词做标签。

**1.2 发展阶段**：时间轴 SVG（图 1，四节点：前两个 accent 色=历史，后两个 gold 色=现在进行时；顶部一行 gold 注释标"行业当前位置"）→ 正文 3 个关键转折（加粗小标题的 li）→ `callout`"理解这个行业的钥匙"（一组对立概念，如"合规的壳 × 灵活的芯"）。年份不确定就写"约 XXXX"并在图注声明推断。

**1.3 业务规模**：4 个 `.stat` 卡（挑最有体感反差的数字）→ 一段解读，落点是"这个行业的基本面是什么"。

## 二、商业模式

**2.1 目标客户与需求**：客户是谁（含层级/渠道结构）→ 需求分层——按行业选一对有张力的切法：组织 vs 经办人（B2B 服务业）、渠道 vs 终端消费者（实体产品业，如影楼要周转和毛利、新人要款式和合身）、决策者 vs 使用者；这对张力往往就是行业存在的原因 → 点出一个"决定成交的隐形事实"。

**2.2 产品与服务**：先一段讲清"这个行业真正卖的是什么"（产品常常只是载体）→ 产品全景图（图 2）。图 2 的三层结构永远回答三问：①由什么构成 ②怎么从头走到尾 ③有哪些形态变体；三层装什么按行业类型选变体：

- **服务/金融型**：服务要素 / 生命周期（"长出来的"最特殊环节 gold 高亮加括注）/ 产品-订单家族（主角 accent 实心）；
- **实体产品型**（制造业，如婚纱）：产品解剖（物理组成 + 成本占比，哪块最贵一眼看出）/ 价值链（原料 → 工艺 → 成品 → 交付，增值最大或卡脖子环节 gold 高亮）/ 品类矩阵（按款型或价格带）。

图后 1–2 段点出产品结构里藏的经营策略。与图 3 的分工：图 2 把产品**拆开看**（物），图 3 把行业**排开看**（角色）——同一个词两图身份不同（"面料"在图 2 是组成部件，在图 3 是上游供应商），且要呼应：图 2 里成本占比最大的部件，往往就是图 3 里话语权最强的上游。

**2.3 上下游与利益分配**（节名用词随行业：金融/服务业叫"资金上下游与利益分配"，制造业叫"产业链上下游与利益分配"，职责不变）：**盈利模式先行**——本节第一个元素必须是「XX 行业怎么挣钱」callout，直白给出"收什么钱 − 付什么钱 = 赚什么"（如"单团毛利 = 团款 − 车房餐票导采购成本"）。不要为此在链条全景图上加标注（图 3 只讲位置与流向，别超载），然后才是：链条全景 SVG（图 3，排版按 SVG 规范的选型规则走竖版或横版双层；gold 标最关键的非常规流向；底部一行注释点出链条里的核心错位）→ 利益分配总述段（每个角色赚什么，加粗角色名）→ 上下游表（列固定为：位置｜角色｜做什么｜核心诉求；位置取值：上游/渠道/中游/下游/协作/监管，按实际取舍）。"主角在链条中的位置"只在这一节回答，不塞进图 2。

## 三、需求分析

**3.1 L1 总流程**：六环节流程图（图 4：主链 6 格，核心环节 accent 实心、对外输出环节 gold；上方 1–2 个胶囊表示外部输入/触发，虚线接入；下方回流虚线标关键反馈循环；底部两行小字标"决策/执行/外部角色"与"业务节奏"）→ 图后一段点出该行业**最反直觉的一点**（加粗），收在"这个行业对任何管理手段的全部要求是……"。

**3.2 分块详解**：4–6 个 `.block`，与图 4 环节一一对应。每块五段（h5 标题固定）：

1. **这个环节在做什么**——业务语言，含该环节的行业黑话（`.term` 就地解释）；
2. **子流程**——`.flow` 步骤链，5 步左右，全部是业务动作；
3. **谁在参与**——业务角色（谁干活、谁决策、谁是外部方），绝无系统和软件商；
4. **业务特征**——频次、量级、时效、复杂度来源；
5. **痛点**——2–3 条，带具体场景与后果。

行业特色图（编号顺延，不止一张也可以）挂在最能体现复杂性的块里（服务/金融业如计息机制图、利润漏斗图；实体产品业如成本结构瀑布、产能与交期示意），示意数字必须标"示意"。灰色环节所在的块加 `warnbox` 合规风险提示。

**3.3 方案切入视角**：`callout`"痛点根因"——把十几个痛点收敛为 2–4 个根因并说明彼此关系（如"前两个决定算不动，第三个决定不敢错"）；根因的比喻和叫法每个行业自拟，不复用其他报告的措辞（同摘要的修辞禁复用规则）→ 有序列表 3 条要点，模式固定：① 这是什么类型的生意（管理手段的价值在哪）；② 业务的原子是什么（一切数据围绕什么归集）；③ 这个行业要的结果是什么（一句话衡量标准）。

## 附录、术语表

`dl.glossary` 分 3–4 组（按主体/交易/工艺与材料/财税/组织等维度，随行业取舍），每组 h4 标题。词条 15–25 个；dt 可带 `<small>` 别名注释；dd 一两句话，讲人话。**词条可点击跳转**：每个词条 div 带 `id="term-主名"`（多别名词条如"短拆 / 过桥"用第一个名字做 id），正文术语写成 `<a class="term" href="#term-主名">` 链接过来（模板已含 `:target` 高亮样式）；正文里没有对应词条的黑话保持 `<span class="term">` 不加链接。

## 页脚

固定要素：访谈来源与时长、"案例数据来自单一 XX，供建立体感，非行业统计口径"、公开资料标注、示意图声明、"本报告为内部学习材料，涉及敏感/灰色内容仅作现状描述"。

## SVG 图规范

- viewBox 宽 760–920，颜色只用 `var(--accent)` `var(--accent-soft)` `var(--gold)` `var(--gold-soft)` `var(--surface)` `var(--ink)` `var(--ink-soft)` `var(--muted)` `var(--line)`；
- 文本 font-size 11–15；每个 svg 带 `role="img"` 和 `aria-label`；
- **文字与任何线/曲线的间距 ≥6px**：注释文字放在曲线的留白侧（下方回流线→文字放线下方），引线宁可缩短也不许扎进字域，长文字先估宽（CJK≈1em/字）再定引线端点；写完跑 `scripts/check_svg_overlap.py` 验证；
- marker 箭头 id 全文件不重复（arr/arrg/fa/pa 各图分开）；
- 语义：accent = 常规/主角，gold = 异常态/现在进行时/钱的流向，虚线 = 非常规或外部流；
- 链条全景图（图 3）中主角块用 accent 实心填充，且必须在块上方加金色标注「▼ 本报告主角」，figcaption 末尾说明"深色实心块为本报告主角（XX）"——读者要像看商场地图一样一眼找到"你在这里"；1.1 的「报告主角与范围」callout 末尾指向该图（"主角在产业链中的位置见 2.3 的图 3"）；
- 链条全景图排版选型：**链条层级 ≥4 层用竖版**（上下游字面意义上就是上和下，位置感一眼到位，两侧标"资金流向 ↓ / 服务交付 ↑"之类的双向箭头；主角若同时是资源调度中心，可在其下方扇形展开资源方）；**层级少但流向复杂（钱/货/票需分层讲）用横版双层**（上层单与货、下层钱与票）；监管/报送类角色用虚线框挂在链条旁。

## 语气与文风

- 解释性写作，读者是聪明但零背景的人；类比要贴生活（"跟房贷提前还款一样"）；
- 一切解释落到具体例子（地名、动作、场景），不写岗位说明书式的抽象职责——图内副标题、正文、表格一体适用（详见 SKILL.md 铁律）；
- 关键结论加粗；不用"令人震惊/势必"等灌水词；
- 引用受访者的生动原话可以保留（如"刷了一年流水不知道挣没挣钱"），加引号。

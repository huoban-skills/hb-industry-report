# 逐节写作指南

每节按此指南填充 template.html。整体基调：**先讲故事建立认知，再讲生意，最后讲业务**；术语首次出现就地解释。五类骨架图必备且固定在对应章节、图号连续编号，骨架之外加不加图由内容形态定——**这两条的完整规则见 SKILL.md（「产出物」骨架 / 「图与文字的分工」）**，本文件只管每节具体怎么写。一图一论点，不为凑数。

**文字要生动**（完整要求见 SKILL.md 「图与文字的分工」，此处不复述）——落到本文件的硬指标：**全文 8000–12000 字为宜**。

**图与正文的关系（全文适用）**：图前一两句话总起本节，图后按图中编号**分条**做案例化讲解（每条加粗动作名）——不许一大段文字堆在一起。**禁止元话语**：不写"拿一单生意把图 3 走一遍""对照图中箭头看""如图所示"，直接讲事，读者自己会对上。图的底注只写**一句题眼**，不写图解说明。

## 封面与摘要

- **主标题**：纯行业名（如"城投定融行业""央国企电商采购供货行业"）。
- **副题**：破折号开头，一句话点出行业本质（"—— 地方隐性债务的形成、兑付与化解"）。
- **目录**：两级——章 + 小节（附录列词条分组）。小节 h3 带 `id="s章-节"`、附录 h4 带 `id="gl-N"`，目录链接与之对应；宽屏（≥1380px）目录固定在左侧栏（无边框轻量样式、细滚动条）并随滚动高亮当前小节，窄屏退化为正文内目录块。模板已含样式与文末脚本，按实际标题生成列表即可。
- **摘要**（`.abstract`，两段、三四句话，宁短勿长）：任务是让人 20 秒明白"这是个什么行业、难在哪、要什么"。这四个要素是暗线，行文以阅读感为主线——基于行业本身自然成文，像向同事口头介绍一个行业那样说话，不套任何固定句式；细节（数字、环节链、机制）都留给正文。

## 章头 lede（每章开头那句话）

固定格式「**本章回答：……？……？……？**」，且必须**逐一对应本章的几个小节**。反例：第一章讲的是"这是啥行业、怎么发展来的、市场规模多大"，lede 却跑去写"服务商处于什么位置"——那是第二章的事。写完回头对一遍小节标题，对不上就重写。

## 一、行业背景

**1.1 行业定义**：一句话定义 → 生活化类比（加粗，这是全文记忆锚点）→ `callout`「报告主角与范围」，**主角先行**：第一句写明"本报告站在谁的视角"（行业链条上的哪个角色是主角，附主体画像），再写讨论范围（含什么业务）、排除项（不含什么），最后带同类主体变体。不要用"行业边界"这类读者看不懂的词做标签。

> 1.2 与 1.3 是全文检索最密集的两节。**检索通则（查什么、查到什么程度停）见 SKILL.md 工作流程第 3 步**，这里只讲这两节特有的取数规则。

**1.2 发展阶段**：时间轴 SVG（图 1，四节点：前两个 accent 色=历史，后两个 gold 色=现在进行时；顶部一行 gold 注释标"行业当前位置"）→ 正文 3 个关键转折（加粗小标题的 li）→ `callout`"理解这个行业的钥匙"（一组对立概念，如"合规的壳 × 灵活的芯"）。

本节特有：**年份必须钉在公开事实上，不许凭素材编**——关键政策/监管节点、标志性事件的真实年份去查（部委文件、行业协会、权威报道），素材里提到的转折（如"市面软件都为批发商设计"）用公开事实坐实或校准。确实查不到精确年份，才写"约 XXXX"并在图注声明推断。

**1.3 业务规模**：4 个 `.stat` 卡（挑最有体感反差的数字）→ 一段解读，落点是"这个行业的基本面是什么"。

本节特有的三条：

- **取最新年度**，卡面标年份。某个最有说服力的指标若最新年度未披露（如利润率只到上一年），沿用最近可得年度并写明"X 年口径，最新未披露"——**不要为凑同一年份而丢掉关键指标**。
- **个案数据放解读段**，明确标"来自单一受访者，非行业统计口径"，不要混进 stat 卡冒充大盘。
- **大盘衬托主角、不喧宾夺主**：大盘（全行业规模/主体数）和主角（某类细分个体）常不是一个颗粒度，要让"大盘极大 × 主角极小/极散/极薄利"的反差替主角说话，而不是把报告写成宏观市场综述。

## 二、商业模式

**2.1 目标客户与需求**：客户是谁（含层级/渠道结构）→ 需求分层——按行业选一对有张力的切法：组织 vs 经办人（B2B 服务业）、渠道 vs 终端消费者（实体产品业，如影楼要周转和毛利、新人要款式和合身）、决策者 vs 使用者；这对张力往往就是行业存在的原因 → 点出一个"决定成交的隐形事实"。

**2.2 产品与服务**：目的是让小白通过 **细分 + 对比** 两件事，看懂这行到底卖什么。图 2 两层，按行业分两类写——**每层装什么随行业变，不套模板**：

- **服务型**：① 提供的是哪几项服务（每项落到客户的一个具体难处：「订单写 A、要 B 照供」「预算没批先送货」）；② 同行业 / 上下游的**服务对比**（央国企比的是四种订单形态，城投比的是四种借法）。
- **实体产品型**：① 产品分哪几类、每类由什么构成（含成本占比，哪块最贵一眼看出）；② 同行业 / 上下游的**产品对比**（按品类、价格带或工艺路线）。

主角的那一种 accent 实心，一眼可辨；对比要落到"为什么这几种不能用同一套经营逻辑"，维度从素材里选，不套固定清单。

**图后的文字，负责把图里的概念讲活**——对图中每个概念做场景化的描述和定义（"点A发B"是什么、"过桥"是什么、逾期之后城投提供的"服务"其实是信访接待和谈判），而不是复述图里已有的结构。

**2.3 上下游业务协作链**：本节只回答一件事——**谁和谁协作、交接什么**。全节不出现金额（钱的事归 2.4）。

结构固定为三块：①**开头一两句话总结本节核心**（这条链的题眼是什么，如"合规的单在平台上过，真实的商机和货在服务商与客户之间直连"）；②**业务协作链图**（见下方「协作链图画法」）；③**图后按图中编号分条讲解** + **上下游角色表**（列固定：位置｜角色｜做什么｜核心诉求；位置取值 上游/渠道/中游/下游/协作/监管，按实际取舍）。"主角在链条中的位置"只在这一节回答，不塞进图 2。

**协作链图画法**（这是被反复返工出来的写法，别自己发挥）：

- **按上下游维度单向排布，从业务起点出发**。顶部一行维度标尺（如「下游 · 渠道 · 中游 · 上游」），主角所在维度标「中游 · 本报告主角」。**起点放在业务真正开始的地方**——央国企电商的起点是客户（于是客户在左、上游在右，是反的），城投定融的起点是政策与化债名单，制造业往往是订单或原料。不要因为"上游习惯画在左边"就把起点挪走。
- **箭头是一单生意走完的顺序**，用 ⓪①②③… 编号（⓪ 留给发生在合规流程之外的起点动作，如"私下透商机"）。箭头上写**具体动作**（报备锁单 / 派单 / 垫资集采 / 灵活送货），不写"要素""交付"。
- **同一维度里实质不同的主体必须拆开**（品牌商能开票报价高 / 本地散户没票报价低），差异写在节点里——那就是行业的题眼。
- **两条线路两种样式**：accent 实线 = 合规的、走明面的协作；gold 虚线 = 绕开明面的灵活链路。底部一行注释点出这条链的核心错位（如"给需求的经办人和走流程的组织，是同一个客户的两张脸"）——**只写这一句题眼，不写图解说明**。
- 主角块 accent 实心填充。

**2.4 资金流转与利益分配**：回答**钱怎么流、这行到底怎么挣钱**（单与货 2.3 讲过，不重复）。四块：

1. **callout 先行**：一句话"收什么 − 付什么 = 赚什么"（如"毛差 = 销售价 − 集采价，但毛差不等于利润"）。
2. **资金流转图，利润账画在图里**：跟一笔具体金额走完（"1 万元订单"），图内一角当场算：毛差 → 扣哪几层 → 真实利润率。不另拆漏斗图（读者看不懂、也接不上钱的流向）。
3. **图后按上下游顺序分条**，和图同方向（起点跟图一致），每条一环：这一环收/付多少、赚的是什么，主角那条落在"层层扣完只剩几成"。加粗引导词**直接用角色名**（「组团社先扣一道」「平台旱涝保收」），**别顶"下游·""上游·"这类维度前缀**——图和角色表已经标了位置，正文再顶就冗余。⚠️ **别用时间线（先掏钱→再等钱→钱到账）**——那是第三章 L1 的活，会和图打架、和第三章重复。
4. **一句话收尾点主角处境**（如"唯一不赚钱、还要凑齐所有人钱的那个"）。

## 三、需求分析

**3.1 L1 总流程**：六环节流程图（主链 6 格，核心环节 accent 实心、对外输出环节 gold；上方 1–2 个胶囊表示外部输入/触发，虚线接入；下方回流虚线标关键反馈循环；底部两行小字标"决策/执行/外部角色"与"业务节奏"）→ 图后一段点出该行业**最反直觉的一点**（加粗），收在"这个行业对任何管理手段的全部要求是……"。

**3.2 分块详解**：4–6 个 `.block`，与 L1 流程图环节一一对应。每块五段（h5 标题固定）：

1. **这个环节在做什么**——业务语言，含该环节的行业黑话（`.term` 就地解释）；
2. **子流程**——`.flow` 步骤链，5 步左右，全部是业务动作；
3. **谁在参与**——业务角色（谁干活、谁决策、谁是外部方），绝无系统和软件商；
4. **业务特征**——频次、量级、时效、复杂度来源；
5. **痛点**——2–3 条，带具体场景与后果。

行业特色图（编号顺延，不止一张也可以）挂在最能体现复杂性的块里（服务/金融业如计息机制图；实体产品业如成本结构瀑布、产能与交期示意），示意数字必须标"示意"。**注意：利润怎么算不在这里画——那是 2.4 资金流转图的活**，别把它挪到第三章来（读者读到 2.4 时就该知道这行怎么挣钱了）。灰色环节所在的块加 `warnbox` 合规风险提示。

**3.3 方案建议**（就叫这个名，不叫"痛点切入视角"）：

- **痛点用一两句话收掉，不列大表**：一个 `callout` 把 3.2 的十几个痛点收敛成 2–4 个死结，说清彼此关系（如"两张皮 → 平台黑箱 → 利润漏损"）；比喻和叫法每个行业自拟，不复用其他报告的措辞。
- **按优先级列价值点，每条落到"哪个角色期望实现什么"**：P0 / P1 / P2 / P3，句式是「**P0 · 老板要"每一单的真实利润看得见"**：……（对应痛点①⑥）」——先写角色和他要的结果，再写怎么实现、对应哪些痛点。P0 那条要说清前提（数据结构建对：业务的原子是什么、一切围绕什么归集）。**开头一句话言明这是"基于访谈与公开资料的建议优先级"**——排序是分析判断，不是受访者确认过的顺序；每条都要能回指 3.2 的痛点或素材依据，受访者没说过的目标（如"老板要逐单看利润"）不许写成他的原话诉求，只能写成"财务希望核算更完整"这类素材撑得住的表述。
- **收尾给一条边界**：这个行业哪里**别碰**（如"前端获客别碰——客情靠人情和线下动作，数字化空间有限"）。有边界的建议才可信。

## 附录、术语表

`dl.glossary` 分 3–4 组（按主体/交易/工艺与材料/财税/组织等维度，随行业取舍），每组 h4 标题。词条 15–25 个；dt 可带 `<small>` 别名注释；dd 一两句话，讲人话。**词条可点击跳转**：每个词条 div 带 `id="term-主名"`（多别名词条如"短拆 / 过桥"用第一个名字做 id），正文术语写成 `<a class="term" href="#term-主名">` 链接过来（模板已含 `:target` 高亮样式）；正文里没有对应词条的黑话保持 `<span class="term">` 不加链接。

## 页脚

固定要素：访谈来源与时长、"案例数据来自单一 XX，供建立体感，非行业统计口径"、公开资料标注、示意图声明、"本报告为内部学习材料，涉及敏感/灰色内容仅作现状描述"。

## SVG 图规范

- viewBox 宽 760–920，颜色只用 `var(--accent)` `var(--accent-soft)` `var(--gold)` `var(--gold-soft)` `var(--surface)` `var(--ink)` `var(--ink-soft)` `var(--muted)` `var(--line)`；
- 文本 font-size 11–15；每个 svg 带 `role="img"` 和 `aria-label`；
- **文字与任何线/曲线的间距 ≥6px**：注释文字放在曲线的留白侧（下方回流线→文字放线下方），引线宁可缩短也不许扎进字域，长文字先估宽（CJK≈1em/字）再定引线端点；写完跑 `scripts/check_svg_overlap.py` 验证；
- marker 箭头 id 全文件不重复（arr/arrg/fa/pa 各图分开）；
- 语义：accent = 常规/合规/主角，gold = 异常态/灵活链路/现在进行时，虚线 = 非常规、绕开明面的流向或票据反向流；
- **主角一眼可辨**：主角块 accent 实心填充；协作链图靠顶部维度标尺标注（「中游 · 本报告主角」，gold 色），资金流转图等其他图在块上方加金色「▼ 本报告主角」。1.1 的「报告主角与范围」callout 末尾指向该图（按名称写"主角在产业链中的位置见 2.3 的业务协作链"，不写死图号）；
- **协作链图、资金流转图一律横版**（画法分别见 2.3 / 2.4，不再提供竖版选型——竖版会丢掉"一单生意走完的顺序"这条主线）；协作链里监管/报送类不在主链上的角色，用虚线框挂在链条旁或只进角色表。

## 语气与文风

- 解释性写作，读者是聪明但零背景的人；类比要贴生活（"跟房贷提前还款一样"）；
- 关键结论加粗；不用"令人震惊/势必"等灌水词；
- 引用受访者的生动原话可以保留（如"刷了一年流水不知道挣没挣钱"），加引号。

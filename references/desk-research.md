# 素材不足时：公开资料检索与留白

适用：用户只给了一句话（"分析一下 XX 行业"）、一个公司名、或几条零散笔记，没有访谈素材。素材介于"充分访谈"和"零素材"之间时，本文件**按节生效**——哪一节素材撑不住，哪一节按这里写。

**没有访谈的报告不是"素材少一点的报告"，是另一件事**：有访谈时产出是"看懂一个行业"，没有时产出是"看清这个行业的公开轮廓 + 知道还该去问什么"。当成前者来写，必然滑向编造。

## 开工前先对齐（必须做，不要跳过）

跟用户挑明三件事，等他确认再动手：

1. **深度会明显弱一档**——行业内部的真实机制（利润怎么被一层层吃掉、灰色环节怎么运转、黑话背后的门道、一线的真实痛点）公开资料里**多半查不到**，查到一线证据的部分是例外（见下"逐节"）；
2. 问他**是先补一手素材，还是接受这个深度**；
3. 接受的话，产出会是**报告（带留白）+ 一份访谈提纲**。

## 检索：先扩口径，再下"资料少"的结论

很多行业没有统一名称和统计口径。**"查不到全国统计" ≠ "没有公开资料"**——把后者当结论写进报告之前，必须先做两层扩展：

- **口径扩展**：不只搜用户给的叫法，并行查行业正式名称、市场俗称、相邻服务名、职业名、监管与采购口径（如"家庭教育陪伴"→"家庭教育指导服务 / 家庭教育指导师 / 家长教育 / 政府购买服务"）。
- **证据类型扩展（渠道六类，SKILL.md 第 3 步的证据地图按此取舍，有无访谈通用）**：

  | 渠道类型 | 拿什么 |
  |---|---|
  | 官方统计与政策（部委、统计局、发布会） | 规模数字、政策年份 |
  | 行业协会 / 披露文件（年报、行业报告、招股书、政府采购公告） | 行业结构、利润率、分配机制、真实买家与金额 |
  | 垂直媒体（开工前先认出该行业的头部垂直媒体是谁） | 行业动态、深度报道、从业者原话 |
  | 司法与监管（裁判文书、监管通报、处罚公告） | 灰色环节、真实纠纷、合规底线 |
  | 企业侧（官网、注册信息、招聘 JD、电商价格） | 主体画像、真实报价、岗位分工 |
  | 学术与调研（论文、地方课题） | 没有官方统计时的替代口径 |

  地方样本、单个项目照写，标"局部观察 / 单一项目口径"，不冒充全国事实。

**控 token 的漏斗**：第一轮广搜只看标题、摘要、机构和年份初筛来源，不逐页精读；筛出真正落进正文的高价值来源（默认 12–18 个）再打开，PDF 只定位相关页；每个来源只记「结论｜年份｜覆盖范围｜来源｜URL」一行，不把网页长文拖进上下文。连续两个新来源没有新增事实就停该分支；第二轮仍补不上的，标成具体的证据缺口（缺的是什么统计、什么样本），不无限搜下去。

## 逐节：什么能写、什么必须留白

同一套骨架，但每节的"资格"不同。**表中所有"查不到——留白"都以做完上面两层扩展仍查不到为前提**：判据是证据质量，不是来源类型——够具体、可溯源的一线公开证据（从业者原话的深度报道、司法判决、监管处罚、招股书/年报、从业者实名分享）查到了就按一手素材对待，标明来源、不冒充访谈；泛泛的二手综述文章不算。

| 节 | 公开资料撑得住吗 | 怎么写 |
|---|---|---|
| 1.1 行业定义 | ✅ 能 | 正常写。类比、主角与范围照常——但主角画像要写"依据公开资料"的那种（区域性中小服务商），不编具体经营细节 |
| 细分行业（按需） | ✅ 能 | 正常写。对比维度与各类角色的公开画像都查得到，公开资料完全撑得住 |
| 发展阶段 | ✅ 能（这是公开资料的强项） | 正常写。政策、监管节点、标志性事件的年份都查得到，且比访谈更准 |
| 业务规模 | ✅ 能 | 正常写。市场规模、增速、渗透率、主体数量——**但没有个案数字来形成"大盘极大 × 主角极薄"的反差**，这个反差是一手素材的招牌，没有就别硬凑 |
| 2.1 目标客户与需求 | ⚠️ 一半 | 客户是谁、组织层面的需求（合规、比价、留痕）能查到；**经办人层面的真实需求（灵活、点A发B）查不到——留白** |
| 2.2 产品与服务 | ⚠️ 一半 | 卖什么品类、有哪几种订单/经营模式，能从公开信息推；**"客户掏差价买的是哪几件事"是访谈才能拿到的，别编**。写不出五件事就老实写模式对比 |
| 2.3 业务协作链 | ⚠️ 一半 | 明面上的链条（谁下单、谁派单、谁供货）能画；**绕开明面的灵活链路（金虚线）查不到——图上不许出现虚构的虚线** |
| 2.4 资金流转与利益分配 | ❌ 大半不能 | 收入来源、平台扣点这类公开条款能写；**"跟一笔 1 万元订单走完四层暗扣、真实利润率 8%"这种账，没有访谈就是编的——留白**。有公开的行业毛利率就引用并标来源，没有就写"无公开数据" |
| 3.1 L1 总流程 | ⚠️ 一半 | 标准业务流程能从公开资料还原；**流程里"最反直觉的一点"往往来自访谈——写不出来就不硬写** |
| 3.2 分块详解 | ❌ 大半不能 | 环节和子流程能写；**「业务特征」和「痛点」是这个 skill 的命根子，且几乎全来自一线——不许用"效率低、信息不透明、依赖人工"这类放之四海皆准的空话填满**。查得到的公开痛点（政策合规压力、行业普遍的账期问题）可以写并标来源，其余留白 |
| 3.3 方案建议 | ❌ | 痛点写不出来，方案建议就是无源之水——**整节留白**，改为指向访谈提纲；3.2 里有据可查的公开痛点足够多时才可少量写，逐条注明依据 |
| 附录 术语表 | ⚠️ 一半 | 公开的行业术语（MRO、集采、供应链金融）能写；**黑话（跑单、点A发B、库存票）查不到——留白** |

**留白怎么写**：在该节位置放一个 `warnbox`，写明「**本节需一手访谈支撑**：公开资料无法回答〈具体是哪几个问题〉，见文末访谈提纲第 N 条」。**不要写成一段读着通顺的空话**——留白必须显眼，让读者一眼看出这里是缺的，而不是被糊弄过去。

## 额外产出：访谈提纲

报告末尾（或同目录下 `<行业名>访谈提纲.md`）附一份提纲，**每一条都由上面的留白反推出来**，不要写成通用问题清单：

- 按上面的表逐节扫，每个留白 → 一组具体要问的问题（"一笔一万元的订单，从平台扣点到最后落袋，中间被哪几层吃掉？各吃多少？"）；
- 标出**谁能回答**（老板 / 销售 / 商务 / 财务）——不同的空缺要找不同的人；
- 按价值排序：先问那些**填上之后整份报告会活过来**的（利润账、真实痛点、灰色链路），而不是按章节顺序问。

这份提纲是无访谈报告真正的交付价值：它让下一次访谈问得准，把"只有公开资料"变成"有一手素材"。

## 铁律在素材不足时的加强

- **「素材忠实度：补的必须真」升级为最高优先级**。有素材时它防的是"填满章节"，没素材时它防的是"整份报告都是脑补"。
- **图内标签仍然必须是具体业务动作**——但公开资料给不出具体动作时，**宁可不画这张图**，也不要用「资源整合」「协同赋能」这类抽象词凑一张看着像样的图。骨架 5 图此时**不是必备**，画不出真东西的那张就留白。
- **数字纪律**：每个数字都必须有公开来源；一个没来源的数字都不许出现（有一手素材时还允许个案数字标口径，这里连这个都没有）。示例测算也要锚定公开条款（如平台公示的扣点比例），没有锚就不画。
- **页脚**必须写明：本报告基于公开资料整理，**无一手访谈**，行业内部机制未经从业者验证，仅供建立初步认知。

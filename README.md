# hb-industry-report

> **别人拿全网二手信息写给行家看，它拿你的一手访谈写给小白看。**
> 把一场两小时的行业访谈，变成一份产品经理能读懂的行业分析报告——从「要素、交付、赋能」里，救出「私下透商机、报备锁单、垫资集采」。

## 你什么时候需要它？

- **刚接一个陌生行业的项目**，手里有一份两小时的客户访谈转写，读三遍还是说不清这行怎么挣钱；
- **要给团队讲透一个行业**，但你不想讲成"产业链上下游协同赋能"——你想讲成"订单写 A、实际送 B，票按订单开、货按真实需求发"；
- **只有一个行业名或一家公司名**，想先摸个底，并且**知道自己还缺哪些非问不可的问题**。

直接问 Agent 会得到一份岗位说明书。这个 skill 的全部价值，就是不让那件事发生。

## 它会交付什么？

一个以行业名命名的文件夹：

```
央国企电商采购供货行业/
├── 央国企电商采购供货行业分析报告.html   # 主交付物，杂志级排版，SVG 内嵌，单文件可分享
├── 央国企电商采购供货行业分析报告.pdf    # 可选交付形态
└── figures/                              # 5 类骨架图的独立 SVG，可复用到飞书/PPT
```

报告固定五段：**摘要 → 行业背景 → 商业模式 → 业务流程 → 术语表**，内嵌 5 类骨架图（发展时间轴、产品服务全景、业务协作链、资金流转、L1 流程）。可选一键同步到**飞书文档**（图固定导成 3200px 宽 PNG 后一起传）。

**报告长什么样**——2.4 资金流转图不只画钱怎么走，还当场跟着一笔 1 万元订单算完账：毛差 2000 → 扣平台点 500、税费 400、资金成本 200、公关 600 → **真实利润 800，利润率 8%（表面毛利看着有 20%）**。这才叫讲清楚"这行怎么挣钱"。

## 绝活：它自带质量闸

方法论 skill 的通病是——规则写了一大堆，没人执行。这个 skill 把能机检的规则**全部做成了脚本**，交付前不过闸，不许导 PDF：

```console
$ python3 scripts/finalize.py 城投定融行业分析报告.html
  FAIL  [章节骨架] 缺 2.3 上下游业务协作链（旧名「资金上下游与利益分配」已废弃）
  FAIL  [术语孤儿] 11 个词条正文从没链过去：专班、承销商、调息、放弃利息…
  FAIL  [图内抽象词] 「兑付业务协作图」出现 要素、交付——图内标签必须是具体业务动作
  FAIL  [图号连续] 图号应为 [1,2,3,4,5,6,7]，实际 [1,2,2,3,3,4,5]（断号/重号）
  WARN  [系统维度用语] 「字段」出现 4 次——报告只写业务，不写系统

总计 FAIL 12 项。交付前必须清零。
```

上面每一条都是从真实报告里抓出来的，不是造的样例。零依赖，纯标准库。


或者直接 clone 到 skills 目录：

```bash
git clone https://github.com/huoban-skills/hb-industry-report.git ~/.claude/skills/hb-industry-report
```

然后把访谈素材丢给 Agent：

> 根据这份访谈，出一份行业分析报告

## 触发方式

- 「根据这份访谈记录，出一份行业分析报告」
- 「帮我梳理一下这个行业 / 把这个行业讲给小白听」
- 「分析一下 XX 行业」（零素材，走公开资料路线）
- 「研究下 XX 这家公司所在的行业」（先跟你确认主角视角，公司 ≠ 行业）
- 「这份录音转写，帮我做成行业普及材料」

## 它和同类有什么不同？

| | 常见的行业研报 skill | hb-industry-report |
|---|---|---|
| 信息源 | 全网二手信息 + 多 agent 并行搜刮 | **一手访谈**（也支持零素材，但会明说深度弱一档） |
| 读者 | 默认你已经懂行（PE / 买方腔） | **零背景的产品经理**，术语两级处理、黑话就地解释 |
| 抽象词 | 「产业链协同」「资源整合」 | **抽象词是黑名单，机检拦截**：要素/交付/赋能/协同一律判 FAIL |
| 质量保证 | 靠 prompt 里写"请注意准确" | **可执行质量闸**，FAIL 不清零不许交付 |
| 产物 | Markdown | 杂志级单文件 HTML + PDF + 独立 SVG + 飞书文档 |

## 安全边界

- **不编内部机制**：没有访谈就拿不到的东西（真实利润账、灰色链路、行业黑话、一线痛点），**宁可留白也不推演**——留白会用醒目的警示框标出来，并附一份"该去问什么"的访谈提纲。
- **数字必须可溯源**：每个数字答不出"这数哪来的"就删；单一案例标"非行业统计口径"，示意测算标"示意"，推断年份标"推断"。
- **灰色环节只描述、不指导**：如实写现状 + 合规风险提示，绝不给操作指引。
- **交付要你点头**：导 PDF / 同步飞书前一定先问，不默认替你决定。

## 文件结构

```
hb-industry-report/
├── SKILL.md                      # 输入分流、工作流程、8 条铁律、自检清单
├── references/framework.md       # 逐节写作指南（含 5 类骨架图的画法）
├── references/desk-research.md   # 素材不足时：哪些能写、哪些必须留白
├── references/lark-sync.md       # 同步到飞书文档（含实战踩坑）
├── scripts/finalize.py           # 终稿统一入口：重导 SVG + 内容检查 + 图形检查
├── scripts/check_report.py       # 质量闸：章节/图号/术语/抽象词/硬编码色
├── scripts/check_svg_overlap.py  # 图形自检：线穿文字、文字出画布
├── scripts/export_png.py         # 飞书插图：SVG → 固定 3200px 宽 PNG
└── assets/template.html          # 设计系统骨架（藏青 × 铜金，深浅双主题）
```

## 验证与测试

改了规则，就拿手头所有历史报告重跑一遍 `finalize.py`——它会重导 `figures/` 并调用两个专项检查器；规则改了而旧报告没跟上，要么该返修报告，要么该反省这条新规则是不是拍脑袋定的。

> 注：本仓库不收录样例报告。真实报告含客户访谈里的业务细节（灰色环节、利润结构），属敏感内容，不进公开仓库。
